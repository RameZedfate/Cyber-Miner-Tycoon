# 可編輯式 PDF 填表工具（AcroForm Form Filler）

針對「**可編輯式 PDF**（PDF 裡有可打字的欄位 / AcroForm）」的理賠申請書，用**欄位名稱**直接灌值。
這是所有填表方式裡**最佳的一種**：不靠座標、不靠 AI、**100% 不跑版**。

> 掃描檔 / 平面 PDF（一張圖、不能打字）**不適用本工具**，要走「座標疊字」的做法。

---

## 一、安裝（只需一次）

```bash
pip3 install pypdf reportlab
```

## 二、三步驟流程

### 步驟 1：看表單有哪些欄位

```bash
python3 inspect_fields.py 空白表單.pdf --json 對應範本.json
```

- 若印出欄位清單 → 這是可編輯式，繼續往下。
- 若顯示「沒有偵測到任何可填欄位」→ 這是掃描檔，改走座標疊字。
- `對應範本.json` 會自動產生一份「欄位名稱 → 空值」的範本。

### 步驟 2：填好資料 JSON

把範本的值填上（key 不要動，那是 PDF 真正的欄位名稱）：

```json
{
  "insured_name": "方翊華",
  "policy_no": "A123456789",
  "accident_desc": "2026-06-10 於自宅樓梯滑倒，左手腕骨折……",
  "claim_accident": "/Yes"
}
```

- 一般欄位填文字即可。
- 勾選框要填它的「勾選值」（用 inspect 查到的，例如 `/Yes`）。
- 「事故說明」這類段落可交給 AI 草擬後貼進來。

### 步驟 3：產出填好的 PDF

```bash
python3 fill_form.py 空白表單.pdf 資料.json 填好的表單.pdf
# 想鎖定不可再改：加 --flatten
python3 fill_form.py 空白表單.pdf 資料.json 填好的表單.pdf --flatten
```

---

## 三、AI 在這裡的角色（重要觀念）

| 工作 | 用 AI 嗎 | 原因 |
|---|---|---|
| 機械式填表 | ❌ 不用 | 確定性程式，每次結果一致、不會跑版 |
| 草擬「事故說明 / 理賠原因」文字 | ✅ 用 | 這才是 AI 真正省時間的地方 |
| 第一次把「你的資料 → PDF 欄位名稱」對應起來 | ✅ 可選 | 有些欄位名是亂碼（如 Text1），AI 對一次存成設定即可 |

---

## 四、已知重點：中文顯示

- 填入的「值」一定存得對。但畫面上中文能不能顯示，**取決於該 PDF 欄位內建的字型**。
- 台灣保險公司的表單通常內建中文字型 → 直接正常。
- 若某張表單中文顯示不出來 → 改用「壓平 + 座標疊字」備援（針對該張另外處理）。
- 建議每張新表單，第一次都先實際填一筆、開來看過再納入正式流程。
